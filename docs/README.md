# 九九マスター - プロジェクトドキュメント

## 📚 概要

このディレクトリには、九九マスターWebアプリケーションの開発に関する全てのドキュメントが含まれています。

## 🎯 プロジェクト概要

**プロジェクト名**: 九九マスター
**目的**: 小学生が楽しく効率的に九九を習得できるWebアプリケーション
**ターゲット**: 小学生（主に小学2〜3年生）
**利用環境**: PC、タブレット、スマートフォン
**技術スタック**: React 18 + TypeScript + Vite + Tailwind CSS

### 実装順序の推奨

1. **フェーズ1: 基盤構築** (約6〜9時間)
   - TICKET-001: プロジェクトセットアップ
   - TICKET-002: 型定義とデータ構造の設計
   - TICKET-003: 九九マスターデータの作成

2. **フェーズ2: コア機能実装** (約15〜19時間)
   - TICKET-004: 共通コンポーネントの実装
   - TICKET-006: 問題画面の実装
   - TICKET-005: レベル選択画面の実装

3. **フェーズ3: 結果・印刷機能** (約9〜11時間)
   - TICKET-007: 結果表示画面の実装
   - TICKET-008: 印刷機能の実装

4. **フェーズ4: UI/UX改善** (約10〜13時間)
   - TICKET-009: アニメーション・エフェクトの実装
   - TICKET-010: レスポンシブデザインの最終調整

5. **フェーズ5: デプロイ** (約2〜3時間)
   - TICKET-011: Vercelデプロイ設定

## 🎨 主要機能

### 7つのレベル

| レベル | 内容 | 問題数 | 目標タイム |
|--------|------|--------|-----------|
| ① | 1〜5のだん（順番・読みあり） | 90問 | 4分30秒 |
| ② | 6〜9のだん（順番・読みあり） | 72問 | 3分30秒 |
| ③ | 81問チャレンジ（順番・読みあり） | 162問 | 6分 |
| ④ | 81問チャレンジ（順番・読みなし） | 162問 | 6分 |
| ⑤ | 81問チャレンジ（ランダム・読みあり） | 162問 | 6分 |
| ⑥ | 81問チャレンジ（ランダム・読みなし） | 162問 | 6分 |
| ⑦ | あなあきチャレンジ（ランダム） | 162問 | 6分 |

- ✅ 問題出題（順番・ランダム・穴あき）
- ✅ タイマー（カウントダウン）
- ✅ 正誤判定
- ✅ 結果表示
- ✅ 間違えた問題の印刷（穴埋め形式）

- ✅ レスポンシブデザイン

## 📝 開発ガイドライン

### コーディング規約
- **命名**: camelCase（変数・関数）、PascalCase（コンポーネント・型）
- **コメント**: JSDocを使用
- **型定義**: `any`は使用禁止
- **インポート**: 相対パスではなく`@/`エイリアスを使用

### コミットメッセージ
```
feat: 新機能追加
fix: バグ修正
docs: ドキュメント更新
style: コードフォーマット
refactor: リファクタリング
test: テスト追加
chore: その他の変更
```

### ブランチ戦略
```
main           # 本番環境
develop        # 開発環境
feature/*      # 機能開発
hotfix/*       # 緊急修正
```

## 🚀 クイックスタート

### 1. プロジェクトのクローン
```bash
git clone <repository-url>
cd Lancers_Matorioshika_MultiplicationApp
```

### 2. 依存関係のインストール
```bash
npm install
```

### 3. 開発サーバーの起動
```bash
npm run dev
```

### 4. ビルド
```bash
npm run build
```

### 5. プレビュー
```bash
npm run preview
```
